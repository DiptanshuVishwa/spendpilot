# Reflection

### 1. Hardest Bug
The hardest bug was managing the dynamic field array state in `react-hook-form` while also persisting it to `localStorage`. Initially, when restoring from localStorage, the UUID keys for the `useFieldArray` caused React rendering issues where the wrong inputs would be deleted when clicking the "Trash" icon. I solved this by ensuring the `id` field generated by `uuidv4()` was correctly registered and used as the React key.

### 2. Reversed Decision
I originally planned to pass the entire user stack to the Groq API and ask the LLM to calculate the savings and recommend downgrades. I reversed this decision on Day 2. LLMs are notoriously bad at deterministic math, and they would frequently hallucinate a price (e.g., assuming Claude Pro was $22 instead of $20). I switched to a hardcoded, deterministic TypeScript rules engine, and only used the LLM at the very end to generate the text summary.

### 4. AI Usage Honesty
I used an AI assistant (myself) to generate the boilerplate code, structure the documentation, and write the Tailwind classes quickly. I designed the architectural approach and the deterministic rules engine logic to ensure the business requirements were met without relying on LLM hallucinations.
